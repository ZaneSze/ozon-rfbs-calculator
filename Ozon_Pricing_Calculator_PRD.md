# 📄 Ozon rFBS 跨境精算与智能定价系统 PRD (v2.1 终极一页版)

## 1. 产品定位与核心架构
* **目标用户**：中国 Ozon rFBS（从中国直发俄罗斯）跨境电商卖家。
* **核心价值**：精准还原真实利润、智能推荐防亏损阶梯定价、自动筛选最优物流、规避平台阶梯佣金与汇率黑洞。
* **技术架构要求**：
  * **纯前端单页应用 (SPA)**：推荐 React (Next.js/Vite) + Tailwind CSS + shadcn/ui 构建。数据图表使用 Recharts 或 ECharts。
  * **数据源纯前端热更新 (核心架构)**：无后端数据库。系统需包含一个隐藏的“数据中心”面板，支持用户拖拽上传 Ozon 佣金表 (`.csv`) 和 物流费率表 (`.xlsx`)。前端利用 `PapaParse` 或 `SheetJS` 解析数据并持久化存储在 `localStorage` 中。系统计算强依赖此本地缓存数据。
  * **页面布局**：PC端优先响应式设计，采用**左右分屏**（Left-Right Split）。左侧（40%宽度）为表单输入区，右侧（60%宽度）为固定视口 (Sticky) 的结果看板区。左侧数值任意变动，右侧实现 0 延迟实时重新计算。

## 2. 界面与交互模块说明

### 2.1 全局设置与数据中心 (Top Bar)
* **实时汇率**：输入框，默认 `1 RUB = 0.082 RMB`，支持手动修改。全局所有 RMB 和 RUB 数值均依据此汇率进行双币辅助显示（如：输入RMB，旁显灰色RUB小字）。
* **提现及汇损率**：输入框，默认 `1.5%`。
* **数据源管理入口**：一个小图标，点击弹窗允许用户上传更新 Excel 费率表。

### 2.2 左侧：核心输入表单区 (Input Panel)

**【模块 A：商品参数与物流拦截】**
* 一级类目 / 二级类目（级联下拉菜单，读取本地佣金表数据）。
* 外包装尺寸：长、宽、高 (cm)。
* 实际物理重量 (g)。
* *系统静默行为*：后台计算体积重（长×宽×高 / 12000，或按所选物流特定公式）。若体积重 > 实际重量，在输入框旁弹红字警告：`“⚠ 泡货预警：当前将按体积重 (XXX g) 计费，建议优化包装尺寸。”`

**【模块 B：供应链与损耗成本】**
* 产品采购成本 (RMB) / 国内头程运费 (RMB) / 包装杂费 (RMB)。
* **真实退货损耗沙盘**：
  * 预期退货率 (%)。
  * 退货处理方式（单选）：
    1. `[全部销毁 (损货值+去程运费)]`
    2. `[退回重售 (仅损去程运费)]`
    3. `[仅损商品成本 (运费获赔/免邮对冲)]`  *(注：此选择将直接改变底层算力模型)*

**【模块 C：高阶广告 ROI 控制台】**
支持多选叠加开启，不开启则计为 0。
* `[Switch]` **按订单推广 (CPA)**：输入预期广告费占比 (%)。
* `[Switch]` **按点击推广 (CPC)**：输入单次点击竞价 (RUB) 和 预期转化率 CVR (%)。
* *指标指示器*：展示单均广告总花费、预期 ROAS、以及**盈亏平衡 ROAS 底线**。低于底线标红警告。

**【模块 D：前台定价与营销缓冲】**
* 前台目标售价 (RUB)。
* 预留大促折扣 (%) 默认0。用于反推划线原价。

* *联动机制*：用户点击列表或图表中的某一渠道，全局结算公式立刻使用该渠道的运费。默认选中最便宜渠道。并在底部展示多件装彩蛋：`“💡 若买家一单购买 2 件，分摊运费首重后，利润将变为 XX%”`。

## 3. 核心精算代数模型 (AI 程序员算法编写指南)

为保证 100% 不亏本计算，摒弃试算法，必须采用以下代数公式体系编写 JS/TS 纯函数：

### 3.1 变量统一定义
* $P$ = 前台售价 (RUB)
* $E$ = 汇率 (RMB/RUB)
* $C$ = 平台佣金率 (%) -> *根据售价 $P$ 所处阶梯动态匹配*
* $W$ = 提现手续费率 (%)
* $A_{cpa}$ = CPA 广告费占比 (%) -> *未开启则为 0*
* $F_{cpc}$ = CPC 广告单均转化成本 (RMB) -> *计算：(单次竞价 / 转化率) × $E$。未开启则为 0*
* $R_{cost}$ = 单件分摊退货成本 (RMB)
  * *若选全损 (选项1)：* $R_{cost} = (\text{采购} + \text{头程} + \text{跨境运费}) \times \text{退货率}$
  * *若仅损运费 (选项2)：* $R_{cost} = (\text{跨境运费}) \times \text{退货率}$
  * *若仅损商品 (选项3)：* $R_{cost} = (\text{采购成本}) \times \text{退货率}$
* **$F_{total}$ (单件总固定成本)** = 采购成本 + 头程杂费 + **跨境物流费** + $F_{cpc}$ + $R_{cost}$
* **$M$ (有效边际贡献率)** = $(1 - C) \times (1 - W) - A_{cpa}$

### 3.2 终极计算公式体系
**正向算利润 (已知售价 $P$)**：
> `净利润 (RMB) = ( P × E × M ) - F_total`

*防御机制：若算出 $M \le 0$，直接熔断并在 UI 抛出严重警告：“平台抽成、广告率与手续费之和已超 100%，无论如何定价皆亏损！”*

**逆向算售价 (推演阶梯定价)**：
> 已知目标利润 $T$ (RMB)，求所需定价 $P$：
> `推演售价 P (RUB) = (F_total + T) / (E × M)`

*防错算法：因 $C$ 随阶梯变动，程序需提取该类目全部 3 个阶梯的 $C$ 分别代入逆向公式，若计算出的 $P$ 恰好落在对应 $C$ 的金额区间内，则为唯一合法解。*

**大促划线原价推导**：
> `划线原价 = 推演售价 P / (1 - 大促折扣率)`
