## 技术改进

### 1. 数据管理优化
- 将默认数据从 `data-hub-context.tsx` 中分离到独立的 `default-data.ts` 文件
- 创建了自动化数据更新脚本 `scripts/update-data.js`
- 支持直接从 CSV 文件解析并生成 TypeScript 常量

### 2. 数据解析增强
- 智能寻找 CSV 表头行
- 支持多种费率格式解析 (¥3.12 + ¥0.0468/1g)
- 正确处理尺寸限制、时效、货值范围等复杂数据
- 自动生成唯一ID避免数据冲突

### 3. 类型安全
- 所有数据严格遵循 TypeScript 类型定义
- 正确处理 Infinity 值 (JSON序列化问题已修复)

## 如何使用更新脚本

如果将来需要更新数据:

1. 准备新的 CSV 文件:
   - 佣金数据文件 (包含一级类目、二级类目、三个阶梯费率)
   - 物流数据文件 (包含评分组、服务等级、第三方物流、配送方式等)

2. 将文件放到项目根目录

3. 运行更新脚本:
   ```bash
   cd ozon-calculator
   node scripts/update-data.js
   ```

4. 脚本会自动:
   - 解析 CSV 文件
   - 生成 `lib/default-data.ts` 文件
   - 输出更新统计信息

5. 验证构建:
   ```bash
   npm run build
   ```

## 注意事项

1. **数据持久化**: 用户上传的数据会存储在 localStorage 中,默认数据作为初始值
2. **数据更新**: 默认数据会在系统初始化时加载,用户上传数据后会覆盖默认值
3. **数据兼容**: 所有数据结构保持向后兼容,不会影响现有功能
