# 本地部署说明

## 一键启动脚本

### Windows 用户

#### 启动服务
双击运行 `start-local.bat` 文件即可

#### 停止服务
双击运行 `stop-local.bat` 文件,或在运行窗口按 `Ctrl+C`

### macOS/Linux 用户

#### 启动服务
```bash
# 首次运行需要添加执行权限
chmod +x start-local.sh

# 运行脚本
./start-local.sh
```

#### 停止服务
在运行窗口按 `Ctrl+C`

## 系统要求

- **Node.js**: v18.0 或更高版本
- **npm**: v9.0 或更高版本

### 如何检查版本

```bash
node -v
npm -v
```

### 如何安装 Node.js

访问 [Node.js 官网](https://nodejs.org/) 下载并安装 LTS 版本

## 脚本功能说明

### start-local.bat / start-local.sh

**功能**:
1. ✅ 自动检查 Node.js 和 npm 是否安装
2. ✅ 显示当前 Node.js 和 npm 版本
3. ✅ 自动安装依赖（首次运行时）
4. ✅ 启动开发服务器
5. ✅ 自动打开浏览器访问 http://localhost:3000

**使用场景**:
- 首次部署项目
- 快速启动本地开发环境
- 不熟悉命令行的用户

### stop-local.bat

**功能**:
1. ✅ 自动查找占用 3000 端口的进程
2. ✅ 强制停止相关进程
3. ✅ 清理残留的后台服务

**使用场景**:
- 服务无法正常停止
- 端口被占用
- 需要强制关闭服务

## 手动启动方式

如果不想使用一键脚本,也可以手动启动:

```bash
# 1. 进入项目目录
cd ozon-calculator

# 2. 安装依赖（首次运行）
npm install

# 3. 启动开发服务器
npm run dev

# 4. 在浏览器打开
# http://localhost:3000
```

## 常见问题

### 1. 端口 3000 被占用

**错误信息**: `Error: listen EADDRINUSE: address already in use :::3000`

**解决方法**:
```bash
# Windows
stop-local.bat

# 或手动查找并停止进程
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```

### 2. Node.js 未安装

**错误信息**: `[错误] 未检测到 Node.js`

**解决方法**: 
下载并安装 [Node.js](https://nodejs.org/)

### 3. 依赖安装失败

**错误信息**: `npm ERR! ...`

**解决方法**:
```bash
# 清理 npm 缓存
npm cache clean --force

# 删除 node_modules 重新安装
rmdir /s /q node_modules  # Windows
rm -rf node_modules       # macOS/Linux

npm install
```

### 4. 浏览器未自动打开

**解决方法**:
手动在浏览器中打开 http://localhost:3000

### 5. Windows 提示"无法运行脚本"

**原因**: Windows 执行策略限制

**解决方法**:
以管理员身份打开 PowerShell,运行:
```powershell
Set-ExecutionPolicy RemoteSigned
```

## 数据持久化

本地运行时,所有数据保存在浏览器的 localStorage 中:
- 佣金数据: `ozon_commission_data`
- 物流数据: `ozon_shipping_data`
- 列映射配置: `ozon_column_mapping`

### 清除数据

在浏览器控制台运行:
```javascript
localStorage.clear()
location.reload()
```
